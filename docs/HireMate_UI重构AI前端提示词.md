# HireMate 前端 UI 重构 AI 提示词

本文档用于辅助前端 UI 生成工具或 AI 设计工具理解 HireMate 的产品结构、页面功能、交互逻辑与设计边界。目标是重塑 HireMate 的整体前端体验，但不能改变产品主流程与业务原则。

## 0. 可直接复制给 UI AI 工具的总提示词

你正在为 HireMate 设计一套全新的招聘初筛系统前端 UI。

HireMate 是一个基于 Python + Streamlit 的招聘初筛工作台，核心用户是 HR、招聘负责人、用人部门面试官和管理员。它不是普通简历列表工具，而是围绕“岗位配置 -> 批量初筛 -> 候选人工作台”建立的结构化招聘辅助系统。

请设计一个专业、清晰、可信赖、适合高频审核工作的 B2B SaaS 风格界面。整体体验要像“招聘运营驾驶舱 + 候选人证据分析工作台”，而不是简单表单堆叠。

必须保持以下产品原则：

- 规则评分器是主判断基础。
- AI reviewer 只是建议层，不能替代人工最终结论。
- RAG / AI / OCR 只能帮助解释、提示风险、补充证据，不能自动点击“通过 / 待复核 / 淘汰”。
- 人工最终决策必须始终清晰可见，并且是最终结果来源。
- 不改变现有五维评分结构。
- 不改变现有数据库 schema。
- 不开放公开注册。
- 管理员功能只对管理员可见。

系统主导航有 3 个核心页面：

- 岗位配置页
- 批量初筛
- 候选人工作台

同时存在辅助区域：

- 登录页
- 管理员账号管理区
- 系统健康检查 / 主流程 smoke 面板
- AI + OCR 环境健康面板
- 单份审核调试入口
- 证据池调试面板

请围绕这些页面重构 UI。所有页面都要有清晰的信息层级、状态提示、操作优先级和错误反馈。界面应支持桌面端为主，同时兼容窄屏浏览。

## 1. 产品定位

### 1.1 产品一句话

HireMate 是一个面向招聘团队的 AI 协同初筛工作台，用规则评分器保证稳定性，用 OCR/RAG/AI reviewer 辅助 HR 更快理解简历、证据、风险和下一步动作。

### 1.2 核心用户

- HR：上传 JD 和简历，批量初筛，进入候选人工作台逐个处理。
- 招聘负责人：查看候选池、风险、推荐理由和人工决策状态。
- 面试官：查看面试建议、关键证据、风险复核点。
- 管理员：管理账号、检查系统健康、检查 OCR/AI 配置。

### 1.3 核心业务流

1. HR 在“岗位配置页”维护岗位、JD、评分权重、硬门槛、AI reviewer 默认配置。
2. HR 进入“批量初筛”，选择岗位，上传多份简历，配置本批次 AI reviewer，执行批量解析和初筛。
3. 系统完成 OCR / 文本提取 / 简历解析 / 规则评分 / 风险分析 / 证据抽取。
4. HR 进入“候选人工作台”，按候选池查看候选人。
5. HR 查看候选人画像、时间线、五维评分、关键证据、AI 建议、风险复核点。
6. HR 做人工最终决策：通过 / 待复核 / 淘汰，并填写备注、优先级。

## 2. 全局设计要求

### 2.1 视觉方向

建议视觉关键词：

- 专业
- 清晰
- 高密度但不拥挤
- 可信赖
- 证据驱动
- 招聘运营感
- 数据看板感

不要做成：

- 纯表单工具
- 炫技型 AI 页面
- 大面积营销落地页
- 只有渐变背景和卡片堆叠的通用模板
- AI 自动决策系统

### 2.2 推荐布局

整体采用：

- 左侧固定导航栏
- 顶部上下文栏
- 中央主工作区
- 右侧或下方详情分析区

桌面端重点优化：

- 批量初筛页面的文件处理流程
- 候选人工作台的左右分栏审核体验
- 证据、风险、AI 建议的可读性

移动端可以降级为单列，但核心操作必须可用。

### 2.3 全局状态颜色建议

颜色语义需要稳定：

- 推荐 / 通过：绿色
- 待复核 / 中风险 / AI 建议人工核验：黄色或琥珀色
- 淘汰 / 高风险 / 失败：红色
- 信息 / 处理中 / AI 建议：蓝色
- 低置信度 / OCR 弱：橙色警示
- 管理员 / 调试：中性灰或深色标签

注意：不要让 AI 建议的视觉权重超过人工最终决策。

### 2.4 信息密度

HireMate 是工作台，不是展示官网。页面可以信息密集，但必须通过分区、标签、折叠面板、状态徽标和固定操作区降低认知负担。

## 3. 页面结构

## 3.1 登录页

### 页面目标

让已有用户安全登录系统。系统不开放公开注册。

### 必备元素

- 产品名：HireMate
- 简短说明：招聘初筛与候选人审核工作台
- 邮箱输入框
- 密码输入框
- 登录按钮
- 登录失败提示
- 当前系统状态可选提示，例如“数据库连接异常时显示启动错误”

### 交互要求

- 不显示注册入口
- 登录失败要给明确但不泄露安全细节的提示
- 登录成功后进入默认页面

## 3.2 全局侧边栏

### 页面目标

提供用户身份、导航、管理员入口和开发调试入口。

### 必备元素

- 当前登录用户信息
  - 姓名
  - 邮箱
  - 角色：管理员 / 普通用户
- 退出登录按钮
- 页面导航
  - 岗位配置页
  - 批量初筛
  - 候选人工作台
- 开发调试入口开关
  - 单份审核调试入口
- 管理员专属入口
  - 账号管理
  - 系统健康检查
  - AI + OCR 环境健康面板

### 权限要求

- 普通用户完全看不到管理员功能。
- 管理员功能要有明确标识，避免误操作。

## 3.3 岗位配置页

### 页面目标

维护岗位库、岗位 JD、评分权重、筛选门槛、AI reviewer 默认配置。

### 页面结构建议

#### A. 岗位首页 / 岗位卡片区

展示所有岗位卡片。

每张岗位卡片包含：

- 岗位名称
- 当前批次数量
- 候选人数
- 通过 / 待复核 / 淘汰数量
- 最近批次时间
- 进入批量初筛按钮
- 进入候选人工作台按钮
- 当前上下文提示：如果该岗位正在被批量页或工作台使用，应标识出来

#### B. 岗位配置管理区

功能：

- 选择已有岗位
- 查看 / 编辑 JD
- 上传 JD 文件
- 更新岗位名称
- 删除岗位
- 保存修改

#### C. 评分设置区

四项基础权重：

- 教育背景匹配度
- 相关经历匹配度
- 技能匹配度
- 表达完整度

要求：

- 总和必须为 1
- 实时显示当前总和
- 不满足 1 时禁止保存
- 提供“自动归一化”
- 提供“恢复模板默认值”
- 提供权重说明弹窗或折叠说明

权重说明文案要直白：

- 权重越高，该维度对总分影响越大。
- 当前总和必须为 1。
- 若不确定，使用模板默认值。

#### D. 硬门槛与风险关注

展示：

- 通过线
- 复核线
- 相关经历最低分
- 技能最低分
- 表达完整度最低分
- 角色模板
- 硬门槛配置
- 风险关注点

#### E. AI reviewer 默认配置

这里只放默认配置，不放运行期开关。

字段：

- 默认 provider
- 默认 model
- 默认能力开关

说明：

- 运行期开关在“批量初筛”页面控制。
- 候选人工作台继承批次级 AI reviewer 配置。

#### F. 历史批次

展示岗位下历史批次：

- 批次 ID
- 创建时间
- 简历总数
- 候选人数量
- 通过 / 待复核 / 淘汰数量
- 打开该批次到候选人工作台

#### G. 新建岗位

字段：

- 岗位名称
- JD 文本
- JD 文件上传
- 创建按钮

## 3.4 批量初筛页

### 页面目标

上传多份简历，执行 OCR / 解析 / 规则评分 / 风险分析 / 证据抽取，并生成批次。

### 页面结构建议

#### A. 当前岗位上下文

展示：

- 当前岗位名称
- 当前 JD 摘要
- 切换岗位入口
- 筛选规则摘要

#### B. 本批次 AI reviewer 设置

字段：

- 启用 AI reviewer
- Provider
  - openai
  - openai_compatible
  - deepseek
  - azure_openai
  - anthropic
  - mock
- Model preset + 自定义 model
- API base
- 直接输入 API Key
- 保存当前账号 API Key
- 是否对新批次自动生成 AI 建议
- 测试 AI 连接按钮

文案要求：

- 不再把 `api_key_env_name` 暴露给普通用户。
- 默认直接输入 API Key。
- 不显示明文 key。
- DeepSeek 默认配置清晰。

#### C. 文件上传区

支持：

- PDF
- DOCX
- TXT
- PNG
- JPG

展示：

- 上传文件列表
- 文件数量
- 单文件提取状态
- OCR / 解析质量
- 是否可进入稳定评估
- 失败原因

#### D. 提取质量预检查

展示：

- OCR 可用状态
- 弱文本数量
- 失败文件数量
- 是否允许弱文本进入初筛
- 对 OCR 弱的文件给出人工复核提示

#### E. 执行批量初筛

主按钮：

- 开始批量初筛

运行中状态：

- 进度条
- 当前处理文件
- 失败/成功统计

#### F. 批次完成卡片

展示：

- 批次 ID
- 总文件数
- 成功进入候选池数量
- 未进入稳定评估数量
- 通过 / 待复核 / 淘汰分流
- 进入该批次候选池按钮

#### G. 分流结果总览

展示：

- 通过候选人
- 待复核候选人
- 淘汰候选人
- OCR 弱质量候选人
- AI reviewer 生成状态

## 3.5 候选人工作台

### 页面目标

这是 HireMate 最核心页面。它应像一个招聘审核 cockpit，让 HR 可以在同一界面完成候选人浏览、证据理解、风险复核、AI 建议查看和人工最终决策。

### 推荐布局

桌面端建议左右分栏：

- 左侧：候选人列表、筛选、排序、批量操作
- 右侧：候选人审核报告、证据、AI 分析、人工决策

### 左侧候选人列表

#### 候选池切换

候选池：

- 待复核候选人
- 通过候选人
- 淘汰候选人

每个候选池显示数量。

#### 快捷筛选

筛选项：

- 全部
- 仅看未人工处理
- 仅看高优先级
- 仅看我处理中
- 仅看他人锁定
- 仅看未领取
- 仅看 AI 建议未生成
- 仅看 AI 建议已生成但未应用
- 仅看 OCR 弱质量 / OCR 能力缺失
- 仅看高风险且待复核

#### 搜索与排序

支持：

- 按姓名搜索
- 按风险等级筛选
- 按处理优先级排序
- 按综合得分排序
- 按风险等级排序

#### 批量操作

支持：

- 勾选当前筛选结果
- 清空当前批次勾选
- 批量标记为待复核
- 批量标记为淘汰
- 批量设置处理优先级
- 批量生成 AI 建议

#### 候选人列表卡片

每个候选人展示：

- 姓名
- 当前候选池
- 自动初筛结论
- 人工最终结论
- 风险等级
- 综合分
- 处理优先级
- AI 建议状态
- OCR 质量状态
- 锁定状态

### 右侧候选人审核报告

#### 1. 候选人概览

展示：

- 姓名
- 文件名
- 解析状态
- 当前候选池
- 自动初筛结论
- 人工最终结论
- 风险等级
- 建议动作

#### 2. 关键时间线

展示：

- 毕业时间
- 最近实习 / 项目时间
- 时间线是否清晰
- 时间线风险提示

#### 3. AI 结构化画像

展示：

- 教育概览
- 实习概览
- 项目概览
- 技能清单
- 岗位族群猜测
- 资历猜测
- 方法 / 产出 / 结果信号

注意：

- 这不是最终结论。
- 若画像置信度低，应显示“建议人工复核”。

#### 4. 解析质量 / 置信度

展示：

- OCR 置信度
- 结构置信度
- 解析置信度
- 分析模式
  - normal
  - weak_text
  - manual_first

弱质量提示：

- 当前 OCR/解析质量偏弱，建议优先人工复核关键信息。

#### 5. 维度代表证据

五维评分结构保持不变：

- 教育背景匹配度
- 相关经历匹配度
- 技能匹配度
- 表达完整度
- 综合推荐度

每个维度展示：

- 分数
- 代表证据
- tags
- 原文片段展开
- 低可读性提示

#### 6. 正向证据 / 反证 / 缺失点

展示三组信息：

- 正向证据：支持推进或支持评分的证据
- 反证 / 质疑点：与岗位要求不完全匹配、证据不足、泛化描述等
- 缺失点：教育、经历、技能、时间线、结果指标等缺失

要求：

- 反证不要写成系统最终淘汰结论。
- 缺失点要像 HR 的复核清单。

#### 7. 关键证据片段摘要

展示：

- source：实习 / 项目 / 教育 / 技能
- tag：JD 命中 / 方法证据 / 结果证据 / 风险证据 / 经历片段
- text：精简证据
- related dimensions：关联评分维度

#### 8. 风险与建议动作

展示：

- 风险等级
- 建议动作
- 风险点列表
- 高风险 / 中风险提示

#### 9. 面试建议

展示：

- 建议追问问题
- 重点核实点
- 面试总结

#### 10. AI 建议采纳状态

展示：

- AI reviewer 状态
- AI reviewer 模式
- AI 建议采纳状态
- 已采纳项

强调：

- AI 建议默认不自动生效。
- 采纳 AI 建议必须由人工确认。

#### 11. 五维评分辅助信息

展示：

- 五维评分卡片
- 评分理由
- 评分 evidence
- 若评分包含已人工确认的 AI 建议修正，显示提示

#### 12. 原始提取与解析信息

展示：

- 解析前清洗稿
- 提取原文
- 原始 OCR 文本
- parsed_resume JSON 或结构化字段

默认：

- 清洗稿优先展示
- 原始 OCR 文本放入折叠区

#### 13. 人工备注与人工决策

这是最终操作区，必须清晰、稳定、视觉权重最高。

操作：

- 设置处理优先级
- 填写人工备注
- 人工最终结论：
  - 通过
  - 待复核
  - 淘汰
- 保存人工决策

要求：

- 人工结论按钮必须和 AI 建议区分开。
- AI 不得自动点击这些按钮。
- 保存后要有成功反馈。
- 候选人应移动到对应候选池。

#### 14. AI 审核建议

展示：

- AI review summary
- score challenge
- risk suggestion
- suggested action
- evidence references
- 应用 AI 建议按钮
- 撤回 AI 建议按钮

要求：

- 默认不自动应用。
- 应用前建议预览影响。
- 应用后仍不等于人工最终结论。

#### 15. 证据池调试面板

仅调试用途。

展示：

- 维度
- 候选证据文本
- source
- tags
- dim_score
- quality
- alignment
- total
- 阈值信息

默认折叠或通过开关显示。

## 3.7 系统健康检查 / 主流程 smoke 面板

### 页面目标

管理员快速确认系统是否能完成“岗位配置 -> 批量初筛 -> 候选人工作台”的最小主流程。

### 检查项

- 数据库连接
- users 表读写
- jobs 表读写
- OCR capability
- AI provider 配置
- 当前是否存在岗位
- 当前是否存在批次
- smoke 脚本结果

### UI 要求

- 用状态徽标展示 pass / warning / fail
- 失败项给出排查建议
- 不显示敏感信息

展示：

- api_base

- 不显示明文 API Key。
- 失败原因要面向用户。

## 4. 关键组件清单

### 4.1 状态徽标

需要统一设计：

- 自动结论：推荐进入下一轮 / 建议人工复核 / 暂不推荐
- 人工结论：通过 / 待复核 / 淘汰 / 未处理
- 风险等级：高 / 中 / 低 / 未知
- AI 状态：未生成 / 已生成 / 已应用 / 已撤回 / 失败
- OCR 状态：正常 / 弱质量 / OCR 缺失
- 锁定状态：我处理中 / 他人锁定 / 未领取 / 过期

### 4.2 评分卡片

五维评分卡片需要：

- 维度名
- 分数
- 简短理由
- 代表证据入口
- 风险或低置信度标记

### 4.3 证据卡片

证据卡片需要：

- source
- tag
- text
- related dimension
- 展开原文
- 低可读性提示

### 4.4 批次卡片

批次卡片需要：

- 批次 ID
- 创建时间
- 简历数
- 候选人数
- 通过 / 待复核 / 淘汰数量
- 打开工作台按钮

### 4.5 文件处理行

每个上传文件展示：

- 文件名
- 文件类型
- 提取方式：文本 / OCR / fallback
- 质量：ok / weak
- 是否可评估
- 错误信息

### 4.6 管理员表格

账号表格需要：

- 搜索或筛选
- 状态标签
- 行级操作
- 操作成功 / 警告反馈

## 5. 交互原则

### 5.1 操作优先级

高优先级：

- 开始批量初筛
- 进入候选人工作台
- 保存人工决策
- 保存岗位配置

中优先级：

- 测试 AI 连接
- 生成 AI 建议
- 应用 AI 建议
- 自动归一化权重

低优先级：

- 调试入口
- 原始 JSON
- 证据池调试
- smoke 结果详情

### 5.2 错误反馈

错误反馈要清楚说明：

- 出了什么问题
- 影响哪个流程
- 用户下一步应该做什么

例子：

- OCR 不可用：提示安装 tesseract / poppler，建议改用 txt/docx 或人工复核。
- AI API 失败：提示 key 缺失、api_base 错误、网络超时或 provider 未实现。
- 数据库初始化失败：提示 backend、cryptography、MySQL 是否启动、schema bootstrap。

### 5.3 AI 安全边界

所有 AI 相关模块必须体现：

- AI 是建议层。
- AI 不能自动改变人工最终结论。
- AI 建议需要人工确认后才应用。
- 应用 AI 建议也只是修正评分/风险/摘要，不等于最终通过或淘汰。

## 7. 页面信息架构建议

### 7.1 用户初次进入系统

默认看到岗位配置页或岗位首页。

推荐顶部展示：

- 当前系统运行状态简短提示
- 当前登录用户
- 当前选择岗位

### 7.2 从岗位到批量

点击岗位卡片的“进入批量初筛”后：

- 批量页自动带入岗位 JD
- 本批次 AI reviewer 配置使用岗位默认值
- 用户可以修改本批次配置

### 7.3 从批量到工作台

批次完成后：

- 显示完成卡片
- 主按钮进入候选人工作台
- 工作台自动选择当前岗位与当前批次

### 7.4 工作台处理候选人

HR 的视觉路径：

1. 看候选池和候选人状态
2. 看候选人概览
3. 看解析质量
4. 看关键证据与评分
5. 看风险与面试建议
6. 看 AI 建议
7. 做人工最终决策

## 8. 设计禁区

不要做：

- AI 自动决定通过 / 淘汰
- 隐藏人工最终决策按钮
- 把 AI 建议设计成比人工结论更醒目
- 把 OCR 弱质量伪装成正常
- 把调试信息默认放在主视觉中心
- 公开注册入口
- 展示明文 API Key
- 展示 password_hash
- 破坏五维评分结构

## 9. 输出给 UI 工具时的建议拆分

如果 UI AI 工具一次处理不了全部页面，建议按以下顺序分批生成：

1. 全局导航 + 登录页 + 基础设计系统
2. 岗位配置页
3. 批量初筛页
4. 候选人工作台主体验
5. AI 结构化画像 / 证据链 / 风险区块
6. 管理员账号与健康检查页
7. 调试面板与边界状态

## 10. 验收清单

### 全局验收

- 三个主页面清晰可达。
- 普通用户看不到管理员功能。
- 页面状态反馈清晰。
- AI 相关视觉不压过人工最终决策。
- OCR 弱质量时有醒目人工复核提示。
- API Key 不明文展示。

### 岗位配置页验收

- 可看岗位卡片。
- 可编辑 JD。
- 可调整四项权重。
- 权重总和不为 1 时禁止保存。
- 可查看历史批次。

### 批量初筛页验收

- 可选择岗位。
- 可上传多份简历。
- 可查看每份文件解析质量。
- 可配置本批次 AI reviewer。
- 可进入候选人工作台。

### 候选人工作台验收

- 可切换候选池。
- 可筛选、搜索、排序。
- 可查看候选人详情。
- 可查看 AI 结构化画像。
- 可查看解析置信度。
- 可查看正向证据、反证、缺失点。
- 可查看风险复核建议。
- 可查看 AI 建议采纳状态。
- 可保存人工最终结论。
- AI reviewer 不会自动替代人工结论。

### 管理员验收

- 管理员可见账号管理区。
- 普通用户不可见。
- 可查看健康检查结果。
- OCR/AI 环境状态可读。
