# HireMate 短期升级任务清单（按优先级）

本清单基于“RAG 降低幻觉”为主线，强调可验证、可回滚、可对比。

## P0（必须先做）

1. 建立正式 benchmark 与对比基线
   - 目标：可复现“无 RAG / 有 RAG”对比数据
   - 产出：
     - 统一 benchmark 数据集
     - benchmark 结果报告（JSON/文本）
   - 验收：
     - `scripts/rag_benchmark.py` 可输出报告
     - 同一数据集重复跑结果一致

2. 历史语料索引扩充
   - 目标：形成稳定的历史语料检索基础
   - 产出：
     - 历史简历 / JD / 证据的索引脚本
   - 验收：
     - `scripts/rag_index_historical_corpus.py` 跑通
     - `vector_store` 中 chunk 总数稳定增长

## P1（次优先级）

3. 批次 / 候选人增量索引
   - 目标：让 evidence grounding 依赖当前批次上下文
   - 产出：
     - 增量索引脚本
     - 批次级触发说明
   - 验收：
     - 新批次导入后索引可更新

4. 轻量 rerank 首轮调参
   - 目标：提高 evidence 命中与可解释性一致性
   - 产出：
     - rerank 权重首轮参数建议
     - 影响范围说明
   - 验收：
     - benchmark 指标有提升或不退化

## P2（可延后）

5. 证据展示一致性校准
   - 目标：减少“证据碎片”与“评分证据”割裂
   - 产出：
     - 证据映射策略说明
   - 验收：
     - 工作台展示更一致

---

## 执行顺序建议

P0-1 → P0-2 → P1-3 → P1-4 → P2-5

- 已开始推进 P0-2：历史语料索引构建可输出报告
