# 任务记录 047｜工作台 cockpit 精修与管理员反馈补全

## 01 任务目标
- 把 React 新前端中的候选人工作台继续向 Stitch 的最终 cockpit 风格靠拢。
- 把管理员页的重置密码、启停账号、管理员权限切换做成完整反馈闭环，而不是只触发基础按钮调用。
- 在不改数据库 schema、不改规则评分主链路、不让 AI 直接替代人工最终结论的前提下完成增强。

## 03 已做决策 + 理由
- 决策：优先精修新 React 前端，不再继续深度打磨 Streamlit UI。
  - 理由：Streamlit 更适合作为 legacy 回滚入口，继续做高保真 UI 收益低、风险高。
- 决策：工作台采用“左侧高密度队列 + 右侧深度审核 cockpit”。
  - 理由：这是 Stitch 设计稿和 DESIGN.md 中最明确的交互结构，也更符合招聘审核场景。
- 决策：管理员页先补完整反馈闭环，再考虑更多权限细节。
  - 理由：当前最影响可用性的不是缺按钮，而是操作后缺少可信反馈。
- 决策：不改后端接口协议，只在前端做更稳定的字段映射和反馈层增强。
  - 理由：最小侵入，方便回滚，也不会打断现有 FastAPI 骨架。

## 05 任务特有约束
- 不改 MySQL / SQLite 业务 schema。
- 不替换规则评分器。
- AI reviewer 仍然是建议层，不能直接替代人工通过 / 待复核 / 淘汰按钮。
- RAG / grounding 继续作为辅助层，不直接输出最终结论。
- 新前端必须复用现有后端与 `src/*` 业务逻辑，不允许重写规则主链路。

## 手工验收建议
1. 运行前端开发环境后进入候选人工作台。
2. 检查左侧是否为队列 rail，右侧是否为 cockpit 式详情布局。
3. 在工作台执行：
   - 保存人工备注
   - 保存处理优先级
   - 通过 / 待复核 / 淘汰
   - 生成 AI 建议
   - 应用 AI 建议
   - 回退 AI 应用
   - 释放锁定 / 管理员强制释放
4. 进入管理员页执行：
   - 新建账号
   - 重置密码
   - 启用 / 停用账号
   - 设为管理员 / 取消管理员
5. 确认每个动作都有明确的处理中、成功或失败提示。
