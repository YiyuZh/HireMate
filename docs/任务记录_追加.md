## 任务记录 035：短期升级方案（RAG 降低幻觉）

### 01 任务目标

- 明确短期升级主线，围绕 RAG 降低 AI reviewer 幻觉与回答不精确问题
- 形成可执行的短期升级方案文档

### 02 当前进展

- 已新增短期升级方案文档：[短期升级方案.md](/D:/apps/HireMate/docs/短期升级方案.md)
- 已明确三段式主线：正式 benchmark、历史语料索引扩充、批次增量索引与 rerank 调参

### 03 已做决策 + 理由

- 决策：短期优先做“benchmark + 历史语料索引 + 增量索引 + rerank”
  - 理由：比继续堆 provider 更直接改善证据 grounding 与解释一致性
- 决策：RAG 只做检索增强与证据层，不改最终结论
  - 理由：保持规则评分器为主链路，避免产品边界失控

### 04 待解决问题

- benchmark 数据集的真实样本覆盖范围与标注标准
- 增量索引触发时机与成本控制

### 05 任务特有约束

- 不改数据库 schema
- 不替换规则评分器
- RAG 不直接产出最终结论

## 任务记录 039：PDF 简历识别修复（强制 OCR + 解析兜底）

### 01 任务目标

- 扫描版 PDF 不再“看起来识别正常但字段为空”
- OCR 无论文本抽取是否“看似可用”都能在低质量时触发

### 02 当前进展

- PDF 文本提取结果若无结构信号将强制进入 OCR
- OCR 低质量时对 2 页以内 PDF 进行更高 DPI 二次尝试
- 解析层增加兜底提取（教育/经历/技能）

### 03 已做决策 + 理由

- 决策：对低结构文本强制 OCR
  - 理由：扫描件常出现“有文本但不可用”，必须强制 OCR 才能恢复
- 决策：解析层兜底补抽
  - 理由：OCR 输出不稳定时，仍尽量给出可用字段

### 04 待解决问题

- 需要更多真实扫描 PDF 做效果回归

### 05 任务特有约束

- 不改数据库 schema
- 不改评分与 AI reviewer 主逻辑
