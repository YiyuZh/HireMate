# 《HireMate》风险分级说明（V1）

> 适用范围：AI 产品经理实习生岗位初筛（单 JD + 单份简历）
>
> 对齐原则：本文件与 `scoring_rules.md` 一致，风险分级用于辅助初筛结论，不替代评分规则与人工判断。

---

## 1. 风险分级定义

### 高风险（High Risk）
表示候选人信息存在明显缺口或关键能力不达标，当前证据不足以支持进入下一轮。

常见特征：
- 关键维度明显偏低（尤其是相关经历、技能匹配度）。
- 简历关键信息缺失严重，导致判断基础不成立。
- 存在较多“不可验证”描述，真实性或可用性不确定。

### 中风险（Medium Risk）
表示候选人有一定匹配基础，但仍有关键问题需要人工核验，不能直接放行。

常见特征：
- 评分处于中间区间，优势与短板并存。
- 关键经历或技能有提及，但证据密度不高。
- 描述不够具体，需通过面试追问补全判断。

### 低风险（Low Risk）
表示候选人整体匹配度较高，信息相对完整，主要做常规核验即可。

常见特征：
- 关键维度达到岗位门槛且表现均衡。
- 推荐理由有明确证据支持（经历、产出、技能应用）。
- 无明显信息断层或高不确定项。

## 2. 每级风险的典型例子

### 高风险典型例子
1. **经历断层**：简历几乎没有与 AI PM 相关的实习/项目，仅有泛校园活动描述。
2. **技能缺口**：JD 要求 SQL/Python/数据分析，简历未体现或仅罗列名词无实践。
3. **信息缺失**：教育信息不完整（缺学历/毕业时间），经历缺时间线和结果指标。
4. **描述不可验证**：大量“参与了很多项目”“做过增长优化”但没有角色、动作、结果。

### 中风险典型例子
1. **相关性一般**：有产品/运营经历，但与 AI PM 岗位的直接关联有限。
2. **技能证据偏弱**：写了关键技能，但在项目中缺少明确使用场景。
3. **表达模糊**：有成果表述但无量化口径，或时间线不够清楚。
4. **单点短板**：总体尚可，但某一关键维度（如相关经历）仅勉强达标。

### 低风险典型例子
1. **经历匹配**：有 AI/数据产品相关项目，能描述“问题-方案-推进-结果”。
2. **技能闭环**：技能不仅列出，还在经历中出现可验证应用（如 SQL 分析支持决策）。
3. **表达完整**：教育、经历、项目、技能结构清晰，时间线连续。
4. **风险可控**：仅存在少量可通过常规面试确认的问题。

## 3. 风险对初筛结论的影响

> 初筛结论类型：
> - 推荐进入下一轮
> - 建议人工复核
> - 暂不推荐

### 高风险 → 结论倾向
- **默认倾向：暂不推荐**。
- 若总分较高但出现高风险项（如关键事实缺失/真实性难判断），应至少降级为**建议人工复核**，不得直接“推荐进入下一轮”。

### 中风险 → 结论倾向
- **默认倾向：建议人工复核**。
- 适合进入“补充验证”流程：围绕风险点生成面试追问，再决定是否放行。

### 低风险 → 结论倾向
- **默认倾向：推荐进入下一轮**。
- 仍保留常规核验，但不因轻微非关键问题改变主结论。

### 与 `scoring_rules.md` 的对齐关系（建议执行）
1. 先按评分规则计算总分与门槛判断。
2. 再应用风险等级做“结论校正”：
   - 出现高风险：结论最多到“建议人工复核”，必要时“暂不推荐”。
   - 出现中风险：结论不高于“建议人工复核”。
   - 低风险：可维持评分结论。

## 4. 适合程序识别的基础规则提示

以下规则用于 `risk_analyzer.py` 等模块做 V1 识别，保持简单可执行。

### 4.1 高风险识别提示（任一满足可判高风险）
- `相关经历匹配度 <= 2`。
- `技能匹配度 <= 2`。
- `表达完整度 == 1`。
- 简历文本过短且关键信息缺失（如教育/经历/技能中至少两类缺失）。
- 出现明显“空泛表述”高频且无结果词（如多次“参与/协助”但无产出）。

### 4.2 中风险识别提示（不满足高风险，且任一满足）
- 总分在中间区间（参考 `scoring_rules.md` 的 60-74）。
- 关键维度仅勉强达标（如相关经历=3 或技能=3）。
- 风险点数量较多（例如 >=2 条）但不构成硬性淘汰。
- 技能命中存在，但“技能-场景-结果”链路不完整。

### 4.3 低风险识别提示（同时满足）
- 总分达到推荐阈值（参考 `scoring_rules.md` 的 >=75）。
- 关键门槛达标（相关经历>=3、技能>=3、表达完整度>=2）。
- 未触发高风险条件，且风险点以常规核验为主。

### 4.4 结论联动建议（伪规则）
```text
if high_risk:
    decision = "暂不推荐" 或 "建议人工复核"
elif medium_risk:
    decision = "建议人工复核"
else:
    decision = 评分规则原结论（通常为"推荐进入下一轮"）
```

---

## 使用建议（HR 场景）
- 风险分级用于“解释为什么需要复核/暂缓”，不是独立替代评分。
- 输出结果建议同时展示：评分、风险等级、风险点证据、面试追问建议。
- 若风险等级与人工判断冲突，以人工复核结论为准。
