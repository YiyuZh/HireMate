## 03 已做决策 + 理由
- 采用 **FastAPI + React/Vite/Tailwind**：
  - 理由：适合把现有 Streamlit 单文件编排逻辑逐步拆为 API + SPA，不需要一次性推翻 `src/`。
- `src/` 保持领域与数据层，不大规模迁移：
  - 理由：当前项目已有较多稳定能力，先通过 `backend/services/*` 包装最稳妥。
- 认证采用 **HttpOnly Cookie + CSRF Header**：
  - 理由：满足“前端不手工保存 access/refresh token”的安全要求，同时便于同域部署。
- 保留 `hiremate-legacy` 容器，不再作为主流量入口：
  - 理由：为线上切流量提供可回滚锚点。
- Vite 配置显式使用真实路径：
  - 理由：当前工作目录是 `D:\HireMate -> D:\apps\HireMate` 的 Junction，构建时会导致 `index.html` 名称越界错误。

## 04 本轮解决过的问题
- **登录页右侧白色大块**
  - 原因：Streamlit 中混用了原始 HTML 容器与后续 widget，视觉上看像一个未真正包裹表单的空白大面板。
  - 处理：改为安全的静态标题块，不再跨 widget 人工包裹 `<div>`。

- **前端构建失败**
  - 报错：Vite build 提示 `index.html` 被解析成 `../../apps/HireMate/frontend/index.html`
  - 原因：当前仓库路径是 Junction，Vite 在 root 和真实路径之间出现混用。
  - 处理：在 `frontend/vite.config.js` 中固定 `root` 为 `realpathSync(configDir)`，并显式指定 `rollupOptions.input`。

- **新前端页面出现乱码/JSX 断裂**
  - 原因：部分页面在编辑过程中混入控制台编码后的乱码文本。
  - 处理：重写 `BatchScreeningPage.jsx`、`WorkbenchPage.jsx`、`JobDetailPage.jsx`、`AdminSystemHealthPage.jsx` 为干净版本。
